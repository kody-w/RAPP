01The Opportunity

Making agents easier to share

Teams across Microsoft are each building great agent tooling in parallel — every team solving a real need, each with its own approach, audience, and learning curve. Agents exist, the work is real, but they don't always travel easily between teams. A shared, lightweight format on top could let that work compound.

What we've been noticing
  • "Yeah, we built that too" — a phrase we've all said. Teams independently solve similar needs because there's no shared baseline to start from.
  • Easier to build than to teach — when a tool is hard to onboard, it stays with its authors, even when the capability deserves a wider audience.
  • The real unlock comes later — what happens after we have agent tooling people can build on together. That's where the AI value compounds.
  • The two-year view — everyone at Microsoft will eventually need a fast way to share agents. A shared format now is cheaper than retrofitting one later.

RAPP is a contribution toward that shared layer — a simple, teachable format that complements the tooling teams already have and lets great work travel, compound, and become accessible to everyone.

02Why RAPP Solves It

One file = one agent. No exceptions.

The sacred tenet

A single Python file contains the documentation, the contract, and the code. A 14-year-old can ship their first agent in minutes. An enterprise can ship the same file to 500,000 seats.

  • 🔁
    Three tiers, zero modification. The same file runs on a laptop (local Flask server), in Azure Functions, and in Copilot Studio on Teams — unchanged.
  • 📦
    Shareable by design. Agents install with a file drop. Registry, store, 7-word speakable phrase.
  • ✅
    Already working. Frozen v1 spec, one-line installer, live agent store, natural-language agent generation.
  • ⚙️
    Engine, not experience. Infrastructure that stays light so capabilities compound — a shared base layer, not another framework to learn.
  • 🤝
    Complements, doesn't replace. RAPP sits alongside the agent tooling teams already use. The goal is to accelerate the great work in flight, not compete with it.

07Agent Management

Enable/Disable Toggle

The brainstem now supports per-agent enable/disable toggles. Files stay on disk — they're just skipped during load. This lets you curate which swarms are active for any given demo or workflow.

How it works

  • 💾
    Backend: .agents_disabled.json in the agents directory tracks disabled filenames
  • 🔌
    API: POST /agents/<filename>/toggle flips state
  • 🔄
    Load: load_agents() skips files listed in the disabled set
  • 🎛️
    UI: Green toggle switch next to each agent in the panel

Visual

execbrief_agent.py — ExecBrief
hacker_news_agent.py — HackerNews
bookfactory_agent.py — BookFactory

One click to toggle. Non-destructive. State persists across restarts.
