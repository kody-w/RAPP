‚ùì Frequently Asked Questions

Common questions answered

What makes RAPP different from other agent frameworks? ‚ñº
RAPP combines rapid prototyping with production-ready features. Unlike basic frameworks, RAPP includes persistent memory, automated setup, beautiful dashboards, and works with any OpenAI-compatible API. It's designed for both quick experimentation and enterprise deployment.
Can I use local models instead of cloud APIs? ‚ñº
Yes! RAPP works with any OpenAI-compatible endpoint. You can use local models via Ollama, LM Studio, or LiteLLM proxy. This gives you complete data privacy and zero API costs while maintaining full functionality.
Do I need to know Python to use RAPP? ‚ñº
No! The setup is fully automated. However, if you want to create custom agents, basic Python knowledge is helpful. The included examples provide templates you can easily modify.
Can I deploy to Microsoft Teams? ‚ñº
Yes! RAPP includes optional Power Platform integration for deploying to Microsoft Teams and M365 Copilot. This is completely optional - RAPP works great as a standalone REST API.
How does memory persistence work? ‚ñº
RAPP uses Azure File Storage to persist conversation context across sessions. Each user gets their own memory space, plus there's a shared memory space for organization-wide knowledge. Memory is automatically managed by the built-in memory agents.
What's included in the ProactiveInsights agent? ‚ñº
ProactiveInsights includes GitHub monitoring, scheduled automation framework, AI-powered trend analysis, automated Monday briefings, persistent memory integration, and a beautiful web dashboard. It's a complete example of production-ready agent capabilities.
