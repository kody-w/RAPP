Later — the horizon

Platform direction

Tier 3: Copilot Studio bundle output plan

Stabilize the Power-Automate-driven guid routing pattern (already shipped at the contract level in v0.12.0). Consider a swarm_factory --target=copilot_studio mode that emits a Power Platform solution bundle directly.

Multi-tenant scale plan

One Tier 2 deployment serves many guids (users / orgs) under shared agents/ + per-user memory namespace. Real-world validation at scale, monitoring, cost tracking.

Richer on-device story plan

IoT, field devices, air-gapped environments. The single-file agent already runs offline with a local LLM; polish the demo story and ship sample agents that showcase it.

Cross-brainstem agent invocation plan

Project-local and global brainstems already coexist on one machine. Smooth the agent handshake so a global brainstem can invoke an agent registered in a project-local sibling, and vice versa, without explicit plumbing.

Principles the roadmap stays faithful to

  • The one-liner stays sacred. Any change must survive curl … | bash on a fresh machine (Article V).
  • brainstem.py stays sacred. Features go in agents/, not in the engine (Article I-A).
  • Three tiers, one model. An agent written on a laptop runs unchanged in the cloud and in Copilot Studio (Article II / Article X).
  • Versions are rollback points. Every release is tagged and immutable; a bad release is one command away from rolling back (Article VIII).
  • No per-user scaffolding. One Tier 2 deployment serves many users via guid routing — not one app per customer.
